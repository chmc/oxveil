Retry Strategy Picker

Choose how to retry a failed phase. Appears as a dialog when clicking "Retry" on a failed phase.

🔄
Standard
Re-run the failed phase from scratch with the same context and instructions.
Context: Full original context Speed: Fast (no preprocessing) Best for: Transient failures, timeouts
✂️
Stripped
Re-run with simplified context. Removes noise that may have caused the failure.
Context: Reduced (error-relevant only) Speed: Medium (context reduction) Best for: Context confusion, hallucination
🎯
Targeted
Re-run with error analysis injected. Feeds the failure output back as guidance.
Context: Original + error analysis Speed: Slow (analysis step first) Best for: Logic errors, test failures
