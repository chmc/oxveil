Form Claudeloop Plan — UX Flow

Superpowers writes a plan → Oxveil detects it in plan preview → user clicks "Form Claudeloop Plan" → the currently viewed plan becomes PLAN.md → ai-parse converts it to phases → ready to execute.

Step 1
Superpowers writes a plan — it appears in Plan Preview
Plan preview already watches docs/superpowers/plans/ and shows files as tabs. The "Form Claudeloop Plan" button is always present — works for any plan format.

Step 1b (alternative)
Auto-notification when new plan stabilizes
If the user isn't watching plan preview, a notification pops up after 5s of file stability. Clicking "Form Plan" does the same thing.

Form it into a claudeloop plan?
Step 2
User clicks button → granularity picker
No file picker needed — the source is the currently viewed plan (or the file from the notification). Just pick granularity for ai-parse.
Phases High-level phases. Each phase is a major unit of work.
Tasks Medium granularity. Each task is a self-contained deliverable.
Steps Fine granularity. Individual steps for detailed tracking.
ⓘ Behind the scenes: source plan content is written to PLAN.md, then claudeloop --ai-parse converts it to ## Phase N: format.
Step 3
ai-parse runs
Progress notification while claudeloop converts the plan.
↻ Forming claudeloop plan...
Step 3b (error)
ai-parse fails or produces invalid plan
If claudeloop --ai-parse fails (network, rate limit, invalid output), show error with retry option. PLAN.md contains the raw source content — user can retry or edit manually.
⚠ Failed to form plan — ai-parse returned invalid output
ⓘ "Retry" re-runs the Form Plan flow (granularity picker → ai-parse) on the same source file. "View Output" opens the output channel for debugging. PLAN.md still exists with raw content — user can also edit it manually and run "AI Parse" separately.
If ai-parse produces a plan but validation fails (e.g. no phases detected), show a different message:
⚠ Plan formed but no valid phases detected. Try a different granularity?
Step 4
PLAN.md opens in editor — sidebar shows Ready
PLAN.md (with ## Phase N: headers) opens in the editor. Plan preview still shows the source plan with the button for re-forming. Sidebar detects PLAN.md and shows "Ready" with Start button.
